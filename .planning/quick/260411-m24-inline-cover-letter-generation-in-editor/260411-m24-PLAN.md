## Pre-flight Findings

**Backend is already done.** No schema or mutation changes needed:

- `convex/schema.ts` line 81: `cvId: v.optional(v.id("cvs"))` already present on `coverLetters` table.
- `convex/coverLetters.ts` line 19: `save` mutation already accepts optional `cvId`.
- `convex/ai.ts` line 223: `generateCoverLetter` action already exists and returns `{ subject, greeting, body, closing }`.

Therefore this quick task is **frontend-only** and splits into 3 atomic commits.

**cvId source:** `EditorPage.tsx` does not currently read the route param (`/editor/:id?`). The hook will accept an optional `cvId?: string` prop, and `EditorPage` will pass `useParams<{ id?: string }>().id` to it. When absent (guest flow, new CV not yet saved), the save mutation is called without `cvId` — which the backend already supports.

**Guest mode:** `saveCoverLetter` requires Clerk auth (`ctx.auth.getUserIdentity()` throws "Unauthenticated"). The drawer must disable the Save button when `user` is not signed in (only Copy/Download available for guests) — matches the existing behavior in `CoverLetterPage.tsx`.

**DS atoms to use (Phase 14 compliance):** `Button`, `Input`, `Textarea`, `Panel`/`PanelHeader`/`PanelBody` from `@/src/shared/ui`. No raw `<button>`/`<input>`/`<textarea>` in the drawer component.

## Must-Haves

- **Truth 1:** User clicks a new "GÉNÉRER_LETTRE" button in section `08. PREVIEW_EXPORT` of the editor's design tab and a drawer slides in from the right without navigating away from `/editor/:id`.
- **Truth 2:** Drawer pre-fills `jobDescription` from the editor state (if set) so the user can generate immediately without re-pasting.
- **Truth 3:** After generation, subject/greeting/body/closing appear as editable textareas; user can regenerate, copy to clipboard, download as `.txt`, and (if signed in) save to Convex with `cvId` link.
- **Truth 4:** Drawer closes via X button, Escape key, or backdrop click; reopening preserves the last generated letter within the same session.
- **Truth 5:** `/cover-letter` standalone page is untouched and still functions for the guest/no-CV flow.

## Risks

1. **`motion` import path** — Task 2 imports from `motion/react`. Verify this is
   the correct subpath for `motion` v12.38.0 (the modern rewrite of
   framer-motion). If not, fall back to plain CSS transitions on a `<div>` with
   `transition-transform duration-300 translate-x-full` / `translate-x-0` toggled
   by `isOpen` — no animation library needed. Decision point during Task 2
   implementation.

2. **`cvId` from route param is string, not `Id<"cvs">`** — The Convex mutation
   types `cvId` as `v.id("cvs")`, so passing a plain `string` may cause a
   TypeScript error at the `saveCoverLetter` call site. Mitigation in the hook:
   cast via `cvId as Id<"cvs"> | undefined` after importing `Id` from
   `@/convex/_generated/dataModel`. If the cast is rejected, a runtime check
   `typeof cvId === 'string' && cvId.length > 0` before passing is acceptable
   since the backend accepts `v.optional(v.id("cvs"))`.

3. **EditorPage.tsx is already oversized** — Adding ~15 lines to a 1800-line
   file nudges it further past the 400-line guideline. This is a pre-existing
   known issue per `STATE.md`; in-scope work is additive only. Extract to a
   separate "ExportActions" sub-component is explicitly out of scope for this
   quick task.
