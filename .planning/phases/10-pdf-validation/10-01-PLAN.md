<objective>
Validate CV text extractability before PDF export and warn users when content may be unreadable by ATS parsers.

Purpose: Since window.print() produces the PDF via browser dialog (we cannot capture the file), we validate the rendered DOM clone that feeds into print. We extract innerText from the iframe clone and compare against expected CV content derived from CVData. If the ratio is too low, the user gets a warning before/during export.

Output: pdfValidation.ts utility, updated pdfExport.ts with validation hook, EditorPage wiring
</objective>

  <name>Task 1: Create pdfValidation utility with tests</name>
  <files>src/features/editor/lib/pdfValidation.ts, src/features/editor/lib/pdfValidation.test.ts</files>
  <behavior>
    - extractExpectedText(cvData: CVData): returns concatenated text from personal_info (name, title, email, phone, location, summary), experience (company, title, bullets), education (institution, degree), skills (skill names), languages (language names)
    - validateCVTextExtractability(renderedText: string, expectedText: string): returns { valid: boolean, ratio: number, warning?: string }
    - ratio = renderedText token count / expectedText token count (split on whitespace)
    - ratio >= 0.6 => valid: true, no warning
    - ratio < 0.6 and > 0 => valid: false, warning in French about degraded text
    - ratio === 0 (empty rendered text) => valid: false, warning about no extractable text
    - Edge case: empty expectedText => valid: true (nothing to validate)
  </behavior>
  <action>
    Create src/features/editor/lib/pdfValidation.ts (under 150 lines):

    1. `extractExpectedText(cvData: CVData): string` — Walks CVData structure and concatenates all user-visible text content. Include: personal_info.name, personal_info.title, personal_info.email, personal_info.phone, personal_info.location, personal_info.summary, each experience's company/title/bullets (only non-hidden displayMode), each education's institution/degree/field, each skill category's skill names, each language's name. Join with spaces. This represents what the ATS should be able to extract.

    2. `validateCVTextExtractability(renderedText: string, expectedText: string): ValidationResult` — Compare token counts. Token = word split on /\s+/. Return { valid, ratio, warning }. Threshold: 0.6 (60%). Warnings in French to match app language:
       - ratio 0: "Le PDF exporté ne contient aucun texte extractible. Les systèmes ATS ne pourront pas lire votre CV."
       - ratio < 0.6: "Le PDF exporté contient peu de texte extractible ({percent}%). Les systèmes ATS pourraient avoir du mal à lire votre CV."

    3. Export `ValidationResult` type: `{ valid: boolean; ratio: number; warning?: string }`

    Write tests FIRST in pdfValidation.test.ts covering:
    - extractExpectedText with full CVData returns concatenated text
    - extractExpectedText with empty CVData returns empty string
    - extractExpectedText skips hidden experiences (displayMode === 'hidden')
    - validateCVTextExtractability with ratio >= 0.6 returns valid
    - validateCVTextExtractability with ratio < 0.6 returns invalid with warning
    - validateCVTextExtractability with empty rendered text returns invalid with zero-text warning
    - validateCVTextExtractability with empty expected text returns valid
  </action>

    **pdfExport.ts changes:**

    1. Add an optional `onValidation` callback parameter to `renderPDF`:
       ```typescript
       export function renderPDF(
         cvElement: HTMLElement,
         designSettings: DesignSettings,
         onValidation?: (result: ValidationResult) => void,
       ): void
       ```

    2. After the iframe content is written (`doc.close()`) but before triggering print, extract text from the cloned DOM element:
       ```typescript
       const renderedText = clone.innerText || clone.textContent || '';
       ```
       If `onValidation` is provided, call it synchronously before the print triggers. The caller decides what to do with the result. Do NOT block the print — the validation is informational.

    3. Move the validation call inside the `iframe.onload` callback, before the `setTimeout` that calls `print()`. Extract text from `doc.body`:
       ```typescript
       iframe.onload = () => {
         if (onValidation) {
           const renderedText = doc.body.innerText || doc.body.textContent || '';
           onValidation needs ValidationResult — import validateCVTextExtractability and call it.
           // But we don't have expectedText here. Solution: pass it as parameter too.
         }
         setTimeout(() => { iframe.contentWindow?.print(); ... }, 500);
       };
       ```

       Actually, simpler approach: change the signature to accept `expectedText` as optional string:
       ```typescript
       export function renderPDF(
         cvElement: HTMLElement,
         designSettings: DesignSettings,
         options?: { expectedText?: string; onValidation?: (result: ValidationResult) => void },
       ): void
       ```
       Inside iframe.onload, if both expectedText and onValidation exist, extract doc.body innerText and call `validateCVTextExtractability(renderedText, expectedText)`, pass result to onValidation.

    **EditorPage.tsx changes:**

    1. Import `extractExpectedText` from `../features/editor/lib/pdfValidation` and `ValidationResult` type.

    2. Update `handleDownloadPDF` and `handlePreviewPDF` to pass options:
       ```typescript
       const handleDownloadPDF = () => {
         if (!cvRef.current) return;
         const expectedText = extractExpectedText(cvData);
         renderPDF(cvRef.current, designSettings, {
           expectedText,
           onValidation: (result) => {
             if (!result.valid && result.warning) {
               notify({ message: result.warning, type: 'error' });
             }
           },
         });
       };
       ```
       Same pattern for handlePreviewPDF.

    3. Keep the notify() call non-blocking — the print dialog still opens. The warning appears in the notification bar after the dialog closes (or concurrently, depending on browser).
