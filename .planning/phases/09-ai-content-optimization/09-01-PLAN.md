<feature>
  <name>Weak Bullet Detection</name>
  <files>src/features/editor/lib/weakBulletDetection.ts, src/features/editor/lib/weakBulletDetection.test.ts</files>
  <behavior>
    Types to export:
    - `WeakBulletIssue`: `'weak-verb' | 'no-metrics' | 'passive-voice' | 'too-short' | 'too-vague'`
    - `WeakBulletResult`: `{ expIndex: number; bulletIndex: number; issues: { type: WeakBulletIssue; label: string }[] }`

    Function: `analyzeWeakBullets(experiences: Experience[]): WeakBulletResult[]`
    - Skips hidden experiences (displayMode === 'hidden')
    - Returns only bullets that have at least one issue
    - Each issue has a French label for tooltip display

    Detection rules:
    - **weak-verb**: Starts with or contains weak verbs: "responsable de", "responsible for", "en charge de", "in charge of", "participé à", "participated in", "aidé", "helped", "géré" (without quantifier), "managed" (without quantifier), "travaillé sur", "worked on"
    - **no-metrics**: No digits anywhere in the bullet (numbers indicate quantified results)
    - **passive-voice**: Contains French passive markers ("a été", "ont été", "était", "étaient") or English ("was done", "were done", "has been", "have been", "was made", "were made")
    - **too-short**: Bullet is less than 20 characters (too vague to be useful)
    - **too-vague**: Contains vague terms without specifics: "diverses tâches", "various tasks", "etc.", "plusieurs projets" without number, "various projects"

    Test cases:
    - "Responsable de l'équipe marketing" -> [weak-verb, no-metrics]
    - "Augmenté le CA de 35% en 6 mois" -> [] (no issues — strong bullet)
    - "A été impliqué dans le projet" -> [passive-voice, no-metrics]
    - "Aidé les clients" -> [weak-verb, no-metrics, too-short]
    - "Géré diverses tâches administratives" -> [weak-verb, no-metrics, too-vague]
    - "Led migration of 3 microservices reducing latency by 40%" -> [] (no issues)
    - "Worked on various projects" -> [weak-verb, no-metrics, too-vague]
    - Hidden experience bullets -> skipped entirely
    - Empty description array -> no results
  </behavior>
  <implementation>
    Create weakBulletDetection.ts (~80-100 lines):
    1. Define WeakBulletIssue union type and WeakBulletResult interface
    2. Define WEAK_VERBS as array of regex patterns (case-insensitive, word-boundary)
    3. Define PASSIVE_MARKERS as array of regex patterns
    4. Define VAGUE_TERMS as array of regex patterns
    5. Create `detectIssues(bullet: string): { type: WeakBulletIssue; label: string }[]` internal function
    6. Create `analyzeWeakBullets(experiences: Experience[]): WeakBulletResult[]` exported function
    7. Labels in French: "Verbe faible", "Aucune metrique", "Voix passive", "Trop court", "Trop vague"

    Follow existing patterns from scoring.ts and keywordAnalysis.ts — pure functions, no side effects, regex-based analysis.
  </implementation>
</feature>
