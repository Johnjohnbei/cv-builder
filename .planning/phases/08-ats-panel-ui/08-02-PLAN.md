    ```typescript
    interface ATSPanelProps {
      score: ATSScoreResult | null;
      keywords: KeywordAnalysisResult;
      hasJobDescription: boolean;
      onAddSkill?: (skill: string) => void;
      onToggleAtsMode?: () => void;
      onRequestAIAnalysis?: () => void;
      isAtsMode?: boolean;
    }
    ```

    The component receives pre-computed data (from useATSAnalysis in EditorPage). It does NOT call the hook itself — it is a pure presentational organism.

    Layout (vertical flow, all in a scrollable container):

    **Section 1 — Score Gauge:**
    - Render `<ScoreGauge score={score.overall} size={120} />` centered
    - Below gauge: "Score ATS" label

    **Section 2 — Sub-scores (3 horizontal bars):**
    - Each bar: label on left, score on right, colored fill bar between
    - Labels: "Format", "Contenu", "Pertinence"
    - For Pertinence: if `score.relevance === null`, show dashed bar with "N/A"
    - Bar fill color: same tier logic as gauge (green >= 75, orange 50-74, red < 50)
    - Bar implementation: div with bg-gray-200 container, inner div with width as percentage and colored bg
    - Extract `getScoreColor(score: number): string` as local helper (same logic as ScoreGauge)

    **Section 3 — Keywords (only when hasJobDescription):**
    - Header: "Mots-cles" with count badge "{matchedCount}/{totalCount}"
    - Render keyword badges in a flex-wrap container
    - Found keywords: green bg (bg-green-100 text-green-800 border-green-300)
    - Missing keywords: red bg (bg-red-100 text-red-800 border-red-300)
    - Each found keyword badge shows location as small tooltip or suffix: "(exp)" "(skills)" etc.
    - Missing keywords: show small "Ajouter" button that calls `onAddSkill?.(keyword)`

    **Section 4 — Suggestions:**
    - Header: "Suggestions"
    - Show top 5 suggestions from `score.suggestions` ordered as-is (already priority-sorted by scoring engine)
    - Each suggestion: text + action button based on content detection:
      - If suggestion mentions "competence" or "skill" -> "Ajouter" button calling `onAddSkill` with extracted skill
      - If suggestion mentions "ATS" and `!isAtsMode` -> "Activer" button calling `onToggleAtsMode`
      - Otherwise -> no button (informational)
    - If more than 5 suggestions, show "Voir tout" toggle to expand/collapse remaining
    - Use local `const [showAllSuggestions, setShowAllSuggestions] = useState(false)` for toggle

    **Section 5 — AI Analysis button (PANL-05):**
    - "Analyse IA complete" button at bottom
    - Calls `onRequestAIAnalysis?.()` — actual AI logic is Phase 9 (deferred per context)
    - Disabled state with tooltip "Bientot disponible" for now (Phase 9 will enable)
    - Use existing Button atom if available, otherwise a styled button matching design system

    **No-JD state (when !hasJobDescription):**
    - Still show Format + Content sub-scores (they work without JD)
    - Show ScoreGauge with partial score
    - Replace Keywords and Pertinence sections with CTA: "Importez une offre d'emploi pour obtenir le score complet et l'analyse des mots-cles"
    - Style CTA as info card (bg-blue-50, border-blue-200, text-blue-700)

    **Null score state (cvData not loaded):**
    - Show simple loading placeholder or empty state
