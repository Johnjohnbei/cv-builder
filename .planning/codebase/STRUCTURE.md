## Directory Purposes

**src/pages/**
- Purpose: Top-level orchestrator components for routes
- Contains: DashboardPage.tsx, EditorPage.tsx, NotFoundPage.tsx
- Key files: 
  - `DashboardPage.tsx` (450+ lines): CV upload, AI generation, saved CV list, admin panel
  - `EditorPage.tsx` (600+ lines): CV editing interface, template selection, design settings, PDF export

**src/features/auth/**
- Purpose: Authentication flows and protected route enforcement
- Contains: Components and utilities for Clerk integration
- Key files:
  - `ProtectedRoute.tsx`: HOC wrapping routes that require authentication
  - `SyncUser.tsx`: Effect hook syncing Clerk user to Convex database
  - `AuthPage.tsx`: Sign-in/sign-up UI

**src/features/editor/**
- Purpose: CV editing, template rendering, AI optimization
- Subdirectories:
  - `components/`: EditorHeader.tsx, EditorNotification.tsx, TemplateConfirmModal.tsx, OverflowIndicator.tsx
  - `hooks/`: useCVLoader.ts, useAutoZoom.ts, useOverflowDetection.ts
  - `templates/`: Template component implementations (A-F) + shared utilities
  - `lib/`: Business logic (displayModes.ts, scoring.ts, pdfExport.ts, autoFit.ts)

**src/features/dashboard/**
- Purpose: Dashboard index/exports
- Key files: `index.ts` (re-exports from pages/)

**src/features/cover-letter/**
- Purpose: Cover letter generation UI
- Key files: `CoverLetterPage.tsx`

**src/features/landing/**
- Purpose: Public homepage and layout wrapper
- Key files:
  - `HomePage.tsx`: Marketing/landing page
  - `Layout.tsx`: Nav bar wrapper component

**src/shared/**
- Purpose: Reusable types, hooks, utilities, and UI components
- Subdirectories:
  - `types/index.ts`: CVData, DesignSettings, Experience, Education, SkillCategory, Language interfaces
  - `ui/`: Button, Input, Select, Textarea, Toggle, Panel, Notification, Spinner, Dropzone, ErrorBoundary, Logo
  - `hooks/`: useAccessCode.ts, useAutoNotification.ts, useDocumentTitle.ts
  - `lib/`: cn.ts (clsx wrapper), export-docx.ts (Word document generation)

**src/lib/**
- Purpose: Application-level utilities (not React-specific)
- Key files:
  - `linkedinParser.ts`: Extract CV structure from LinkedIn PDF exports
  - `pdfTextExtract.ts`: PDF text extraction using pdfjs-dist
  - `utils.ts`: General helper functions

**convex/schema.ts**
- Purpose: Database schema definition
- Tables: users, cvs, coverLetters, accessCodes, accessRequests
- Key validators: personalInfoValidator, educationValidator, languageValidator, designValidator
- Note: experience and skills use v.any() because AI responses have unpredictable fields

**convex/cvs.ts**
- Purpose: CV persistence and retrieval
- Functions:
  - `listMyCVs` (query): Returns user's saved CVs (up to 50, ordered descending)
  - `createMyCV` (mutation): Insert new CV with full structure
  - `getById` (query): Fetch specific CV with authorization check
  - `remove` (mutation): Delete CV with ownership verification

**convex/users.ts**
- Purpose: User profile management and lastGeneratedCV caching
- Functions:
  - `getMe` (query): Fetch current user's profile
  - `store` (mutation): Create or update user record from Clerk identity
  - `updateLastGeneratedCV` (mutation): Cache the last AI-optimized CV in user profile

**convex/ai.ts**
- Purpose: AI-powered CV optimization and bullet improvement
- Provider abstraction:
  - Primary: NVIDIA NIM (meta/llama-3.1-70b-instruct)
  - Fallback: Gemini via OpenAI-compatible API
- Functions:
  - `optimizeCVForPage` (action): Generate structured CV from PDF/JSON + job description
  - `improveBulletPoint` (action): Enhance a single bullet point
- Helpers: getProvider(), getClient(), getModel(), chatJSON(), safeParseJSON()

## Naming Conventions

**Files:**
- Components: PascalCase (Button.tsx, EditorPage.tsx)
- Hooks: camelCase prefixed with `use` (useAutoZoom.ts, useAccessCode.ts)
- Utilities: camelCase (scoring.ts, displayModes.ts, cn.ts)
- Tests: .test.ts or .spec.ts suffix (scoring.test.ts)

**Directories:**
- Features: kebab-case (editor, cover-letter, landing)
- Utilities: lowercase (lib, shared, pages)

**React Components:**
- Named exports for hooks and utilities
- Default export for page/feature components
- Props interfaces suffixed with Props (EditorPageProps)

**Convex Functions:**
- Query: lowercase verbs (getMe, listMyCVs, getById)
- Mutation: uppercase first letter (store, updateLastGeneratedCV, remove)
- Action: camelCase verb phrases (optimizeCVForPage, improveBulletPoint)

**TypeScript Interfaces:**
- Exported types use PascalCase (CVData, DesignSettings, Experience)
- Internal/local types lowercase (cvLoaderResult, aiProvider)

## Where to Add New Code

**New Feature (e.g., "Skills Assessment"):**
- Primary code: `src/features/skills-assessment/components/SkillsPage.tsx`
- Tests: `src/features/skills-assessment/skills.test.ts`
- Hooks: `src/features/skills-assessment/hooks/useSkillsData.ts`
- Types: Add to `src/shared/types/index.ts` or feature-local types file
- Routing: Add Route in `src/App.tsx`, lazy-load page

**New Component/Module (shared):**
- Implementation: `src/shared/ui/NewComponent.tsx` (if UI atom/molecule)
- Export: Add to `src/shared/ui/index.ts`
- Use cases: Available everywhere via `import { NewComponent } from '@/src/shared/ui'`

**New Template:**
- Implementation: `src/features/editor/templates/TemplateG.tsx`
- Export: Add to `src/features/editor/templates/index.ts`
- Register: Add entry to `TEMPLATE_MAP` in `src/features/editor/templates/CVRenderer.tsx`
- UI: Add option in EditorPage template selector

**New Utilities:**
- Shared helpers: `src/shared/lib/newHelper.ts`
- Feature-specific: `src/features/editor/lib/newHelper.ts`
- Export: Add to `index.ts` in same directory

**New Convex Function:**
- Database operations: Add query/mutation to `convex/cvs.ts` (or new file if different entity)
- Serverless logic: Add action to `convex/ai.ts` or new file
- Schema updates: Edit `convex/schema.ts`, then auto-generated types in `_generated/` update

## Special Directories

**src/shared/ui/:**
- Purpose: Reusable UI components (atoms/molecules)
- Generated: No (hand-written)
- Committed: Yes
- Strategy: Design system approach; components use Tailwind tokens only, no hardcoded colors

**convex/_generated/:**
- Purpose: Auto-generated type definitions from schema and functions
- Generated: Yes (auto by Convex CLI on `convex deploy` or dev)
- Committed: Yes (safe to commit; regenerated on each deploy)
- Do NOT edit manually; changes come from modifying convex/ source files

**dist/:**
- Purpose: Build output
- Generated: Yes (by `npm run build`)
- Committed: No (in .gitignore)
- Cleanup: Run `npm run clean` to remove

**public/:**
- Purpose: Static assets served at root URL
- Generated: No
- Committed: Yes
- Example: favicon.ico, social images

**node_modules/:**
- Purpose: Installed dependencies
- Generated: Yes (by npm install)
- Committed: No (in .gitignore)
